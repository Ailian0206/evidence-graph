# C1 工作台验收修复设计

修订日期：2026-07-24
状态：用户已确认

## 1. 目标

修复 C1 本地验收中发现的登录反馈、用户身份、研究运行状态、研究表单和页面导航问题，并明确公开 Evidence Graph 产品介绍与登录后研究工作台的职责。用户应能从公开站点理解产品、进入真实工作台、识别当前账号、浏览项目和报告，并在任意研究子页面返回上一级。

## 2. 信息架构

- `/{locale}/evidence` 保留为公开的 Evidence Graph 产品介绍和确定性示例入口，主导航名称改为“产品介绍 / Product overview”。
- `/{locale}/app` 是登录后的真实研究工作台，公共主导航名称改为“进入工作台 / Open workspace”。
- 工作台提供“研究项目 / Research projects”和“研究报告 / Research reports”两个稳定入口。
- `/{locale}/app/reports` 新增当前用户的报告列表；报告仍归属于研究项目，不改变数据库结构。
- 真实研究详情继续使用 `/{locale}/app/research/{id}`，公开报告继续使用 `/r/{slug}`。

## 3. 登录与用户身份

GitHub OAuth 保持整页重定向，不实现弹窗协议。GitHub 已授权过应用时可能直接返回，不保证重复显示授权确认页。登录按钮提交后立即进入 pending 状态，禁用重复提交并显示“正在前往 GitHub / Redirecting to GitHub”。OAuth 错误继续回到本地化登录页并展示稳定错误。

登录后的工作台头部展示经过收窄处理的账号信息：优先使用 GitHub 用户名，其次使用邮箱，最后使用稳定的通用账号文案。工作台同时提供退出登录命令。应用不把完整 OAuth metadata 传入客户端。

## 4. 工作台与报告列表

新增工作台级导航组件，负责项目入口、报告入口、当前账号和退出登录。它只出现在受管理工作区；公开产品介绍与公开报告不展示登录后导航。

报告列表读取当前用户通过 RLS 可访问的报告及所属项目信息，按创建时间倒序展示：

- 报告对应的项目标题和研究问题。
- 报告版本与草稿、已发布或已撤销状态。
- 创建或发布时间。
- “打开研究”始终回到所属研究详情。
- 已发布且具有 slug 的报告额外提供“打开公开报告”。

空列表显示明确空状态和“新建研究”入口。查询失败使用稳定错误页，不泄漏数据库错误。

## 5. 运行反馈与表单交互

- `queued` 和 `running` 研究状态的加载图标持续旋转；尊重 `prefers-reduced-motion`，减少动画时保持静态。
- 创建研究提交期间，按钮显示加载图标与进行中文案，避免重复提交。
- 输入控件聚焦时只显示一个清晰焦点边界，不叠加可见的 border 与 outline。
- 来源区域允许标题和“添加来源链接”按钮在窄宽度下换行，按钮文案不裁切、不溢出。
- 新建研究页、排队页、运行页、失败页和研究结果页都提供返回项目列表的页面内入口，不依赖浏览器返回按钮。

## 6. 数据与组件边界

- Auth server session 负责从 Supabase User 中提取最小账号摘要。
- App workspace navigation 只接收账号摘要和 locale，不直接读取 Supabase。
- Report list store 负责查询和映射当前用户报告摘要；页面负责鉴权和组装依赖。
- 现有项目、研究、报告发布和公开报告契约保持不变，不新增 migration。
- Demo 研究继续公开可用，不因工作台导航而增加登录要求。

## 7. 测试与验收

所有行为按 RED-GREEN 执行：

- Auth 单测覆盖 OAuth pending 反馈和账号摘要回退顺序。
- 工作台 UI 单测覆盖项目/报告导航、账号信息、退出登录和返回入口。
- Report store/page 单测覆盖归属过滤、排序映射、草稿与公开报告入口。
- 研究状态单测覆盖 `queued/running` 的动画标记。
- 表单单测与视觉测试覆盖提交 loading、单层焦点样式和来源按钮换行。
- E2E 覆盖受保护的报告列表路由和双语入口。
- 在 390x844、1024x768、1440x1000 检查无横向溢出、裁切和重叠。
- 完成聚焦测试后运行 `npm run test:managed`；例行验证不调用真实 Provider。

## 8. 明确不做

- 不实现 OAuth popup 或强制 GitHub 每次重复授权。
- 不修改数据库 Schema、RLS、Provider、Production 或部署配置。
- 不实现报告搜索、筛选、导出、分页或跨用户公开报告目录。
- 不开始 C2-C6，不合并 Draft PR #18，直到用户重新验收并完成独立 Claude Code 审核。

## 9. 完成标准

用户能区分公开产品介绍与真实工作台；登录操作有即时反馈；登录后能识别当前账号；项目和报告均有明确入口；研究生成有可见 loading；表单无双层焦点边框且文案可换行；所有研究子页面可通过页面内入口返回项目列表。
